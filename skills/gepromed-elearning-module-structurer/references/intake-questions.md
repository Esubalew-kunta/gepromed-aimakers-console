# E-learning Module Structurer — intake questions (FULL tier)

Ask these **once, batched, at most 5**, each with a default/options so the
designer answers in seconds. If the brief already answers a question, **do not
ask it** — infer and state the assumption. Offer the escape hatch. Cap at 2 rounds.

> Reply `go` and I'll proceed with the defaults below.

1. **Specialty / topic?**
   - Vascular surgery · ophthalmology · other (specify) — and the specific topic/procedure.
   - *Default: infer from the brief; if only a broad area is given, scope to one coherent topic.*

2. **Target learners & level?**
   - Residents · qualified surgeons/HCPs · mixed — and level (foundational / intermediate / advanced).
   - *Default: residents at intermediate level (drives Bloom level of objectives).*

3. **Learning objectives (or the topic to derive them from)?**
   - List the outcomes the learner must achieve, or give the topic and I'll draft Bloom-aligned objectives.
   - *Default: derive 1 module-level objective per module from the topic, action-verb, at the learner level.*

4. **Number of modules + total duration?**
   - e.g. "3 modules, 6 hours total" — and chapters per module if you have a preference.
   - *Default: 3 modules, ~3–5 chapters each, total duration [bracketed] if unspecified; ~30–60 min/module guideline.*

5. **Assessment type + prerequisites?**
   - Assessment: MCQ / case-based quiz / practical/simulation checklist / mixed.
   - Prerequisites: required prior knowledge, or none.
   - *Default: case-based quiz per module + a short knowledge check per chapter; prerequisites = the stated learner level.*

---

**When to skip intake entirely:** the brief already specifies specialty, learners
& level, objectives (or a clear topic), module count/duration, and assessment
type. In that case, state a one-line `Assumptions:` and structure the course.

**Clinical-content note:** this skill structures the *learning architecture*. It
does not author clinical content. Any specific clinical fact, protocol, dose,
duration, or outcome value is a placeholder `[bracketed]` for the subject-matter
expert to supply and validate — never invented.
