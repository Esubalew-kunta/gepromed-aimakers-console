# GEPROMED email-type templates

Skeletons for the most common GEPROMED emails. Pick by `email_type`, then fill
with real facts (bracket unknowns), apply the recipient playbook and house voice.
These are scaffolds, not scripts — adapt naturally, never sound boilerplate.

Legend: `[…]` = placeholder the sender confirms.

---

## reply (réponse)
**Goal:** answer clearly, resolve the thread, state the next step.
```
Subject: Re: [original subject]
[Salutation],
[One-line acknowledgement of their message.]
[Direct answer / the information requested.]
[Next step or your ask, with a date if relevant.]
[Closing],
```

## relance (follow-up)
**Goal:** prompt a response without pressure; restate context + a soft deadline.
```
Subject: [Relance / Follow-up] — [topic] (réf. [ref])
[Salutation],
Je me permets de revenir vers vous concernant [topic], évoqué le [date].
[One line of context / what is pending.]
Pourriez-vous me confirmer [the ask] d'ici le [date] ?
[Closing],
```

## invitation
**Goal:** invite to a training/event/meeting; give the essentials + a clear RSVP.
```
Subject: Invitation — [event] · [date]
[Salutation],
Nous serions heureux de vous convier à [event], qui se tiendra le [date] à [lieu].
[1–2 lines: what it is, why it is relevant to them, tied to patient safety / the pillar.]
[Practical: programme, horaires, lien d'inscription [placeholder].]
Pourriez-vous nous indiquer votre disponibilité d'ici le [date] ?
[Closing],
```

## thank-you (remerciement)
**Goal:** sincere thanks tied to mission; never transactional.
```
Subject: Merci — [context]
[Salutation],
Merci pour [contribution / participation / partenariat].
[One line on the concrete value / impact, tied to patient safety.]
[Optional: next step or continued collaboration.]
[Closing],
```

## decline (refus / réponse négative)
**Goal:** say no respectfully, keep the relationship, offer an alternative if possible.
```
Subject: Re: [request]
[Salutation],
Merci pour votre proposition / demande concernant [topic].
Après examen, nous ne sommes pas en mesure de [do X] [for reason, if appropriate].
[Optional alternative: "En revanche, nous pourrions [alternative]."]
[Closing],
```

## scheduling (prise de rendez-vous)
**Goal:** propose/confirm a time efficiently.
```
Subject: [Rendez-vous / Call] — [topic]
[Salutation],
Pour avancer sur [topic], seriez-vous disponible [option 1] ou [option 2] ?
[Optional: call link / location placeholder.]
Je m'adapte à vos contraintes.
[Closing],
```

## info-request (demande d'information)
**Goal:** ask precisely for what you need, make replying easy.
```
Subject: Demande — [what]
[Salutation],
Dans le cadre de [context], pourriez-vous me transmettre [specific items, listed]?
- [item 1]
- [item 2]
Cela nous permettra de [why]. Idéalement d'ici le [date].
[Closing],
```

## bad-news / sensitive (annonce délicate)
**Goal:** clarity + tact; never defensive; route for validation first if regulated.
```
Subject: [topic]
[Salutation],
[Direct, factual statement of the situation — no euphemism, no blame.]
[What it means / impact, only what is known. Unknowns bracketed.]
[What happens next / what you propose.]
[Closing],
```
> If the content touches contracts, pricing, patient data, or regulatory claims,
> add a Note flagging the responsible role (RQ / DPO / RAF / Direction) to review
> before sending.

---

### Common bilingual building blocks
- Acknowledgement — FR: « Merci pour votre message. » EN: "Thank you for your message."
- Soft ask — FR: « Pourriez-vous me confirmer … ? » EN: "Could you confirm …?"
- Availability — FR: « Je reste à votre disposition. » EN: "I remain at your disposal."
- Deadline — FR: « idéalement d'ici le [date] » EN: "ideally by [date]".
