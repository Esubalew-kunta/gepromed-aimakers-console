# GEPROMED LinkedIn Post Drafter — intake questions (FULL tier)

Ask **one batched round of at most 5** numbered questions before drafting. Each
carries a suggested **default** or 2–3 options so the user answers in seconds.
Always offer the escape hatch. **Skip any question the brief or `MEMORY.md`
already answers.** Cap at 2 rounds, then proceed on stated assumptions.

---

## The batched set (ask these, ≤5)

> A few quick questions so the post lands right (reply `go` to take all defaults):
>
> 1. **Format?** text post · carousel · image post · event (announcement/recap) ·
>    scientific-publication. *(default: text post)*
> 2. **Voice?** the GEPROMED **company page** · a **role-based expert voice**
>    (e.g. "Test Platform lead", "clinical-research engineer") — never a real name.
>    *(default: company page)*
> 3. **Primary audience & goal?** audience = surgeons / manufacturers / researchers
>    / institutions / participants / partners / general HCP; goal = awareness /
>    registration / recruitment / credibility / thanks. *(default: general HCP, awareness)*
> 4. **Which proof points / facts may I use, and any date or link?** Paste the true
>    figures, the event date, or the publication citation/DOI. *(default: none —
>    I will bracket everything unverified and invent nothing)*
> 5. **CTA & language?** CTA = register / read the paper / contact us / none;
>    language = FR / EN. *(default: no hard CTA; mirror your message's language)*
>
> Reply `go` and I'll proceed with the defaults above.

## Notes for the model
- If `format` = **event**, make sure question 4 surfaces the **event date** and
  **location**; bracket them if unknown — never guess a date.
- If `format` = **publication**, make sure question 4 surfaces the **citation/DOI**
  and the **one-line plain-language takeaway**; never invent results.
- If `voice` = a role, confirm it is a *role*, not a real individual. The skill
  never publishes as a named person.
- Defaults exist so the skill never stalls. State each inferred choice in the
  `Assumptions:` line of the output.
- Never re-ask anything in the user's brief or in `MEMORY.md` (e.g. a standing
  hashtag set or a default voice).
