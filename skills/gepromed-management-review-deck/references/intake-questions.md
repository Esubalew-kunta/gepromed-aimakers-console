# Management Review Deck — intake questions (Intake tier: FULL*)

This skill is **mode-dependent**:

- **Deck mode → FULL.** Building the §9.3 presentation needs the period and the
  source figures, so run a **structured intake before building** (below).
- **Minutes mode → MINIMAL.** Writing the compte rendu from supplied notes is
  self-defining: **skip this intake entirely**. Declare *"Intake tier: MINIMAL —
  writing from your notes"*, structure the notes per `references/minutes-format.md`,
  and ask only on genuine ambiguity (e.g. unclear language). Bracket anything
  missing rather than asking a full round.

---

## Deck-mode intake (ask all at once, ≤5 questions)

Open with:

> **Mode: Deck · Intake tier: FULL — five quick questions so the slides match the
> real review, then the RQ validates the figures. Reply `go` and I'll build the
> standard ISO 9001 §9.3 deck with every KPI and result bracketed for the RQ.**

1. **Period covered?** e.g. "année 2025", "janvier–décembre 2024", "S1 2025".
   *(default: `[période — à confirmer]`.)*

2. **Which sections / KPIs to include?** Use the standard §9.3 set (see
   `references/iso-9001-9.3.md`) or list your own. Common GEPROMED KPIs: training
   satisfaction, conformity rate, processing/turnaround time, NC count, audit
   results, objectives status.
   *(default: full standard §9.3 input/output set, KPIs bracketed.)*

3. **Source documents for the figures?** Where do the numbers come from — e.g.
   satisfaction survey, NC register, internal audit report, KPI dashboard? (Names
   only; the skill does **not** read values you don't paste in.)
   *(default: note "[figures from RQ source documents — to supply]".)*

4. **Audience?** Direction / comité qualité / external auditor / mixed. This sets
   the tone and the level of context on each slide.
   *(default: Direction + comité qualité, internal register.)*

5. **Language and output filename?** FR or EN; desired `.pptx` filename.
   *(default: mirror the request; filename `revue_direction_[période].pptx`.)*

## On `go`
If the user replies `go`, do **not** stall: build the **standard §9.3 deck** with
every section present and **every KPI / result / decision bracketed** for the RQ,
plus the VALIDATION FLAG. A complete bracketed deck the RQ can fill is the goal —
never an invented figure. Then generate the `.pptx` with the bracketed content so
the RQ sees exactly which slides need numbers.

## Never ask
- Anything `MEMORY.md` already answers (e.g. the standard KPI names or process list).
- For the actual KPI *values* as a blocking question — if not pasted in, bracket
  them and let the RQ fill the deck.
