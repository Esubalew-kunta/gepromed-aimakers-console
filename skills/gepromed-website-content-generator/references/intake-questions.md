# Website Content Generator — intake questions (FULL tier)

Ask these **once, batched, at most 5**, each with a default/options so the writer
answers in seconds. If the brief already answers a question, **do not ask it** —
infer and state the assumption. Offer the escape hatch. Cap at 2 rounds.

> Reply `go` and I'll proceed with the defaults below.

1. **Page type?**
   - a) Homepage
   - b) Training page (course — Qualiopi fields required)
   - c) Authority page (explant analysis · vascular devices · R&D/testing · clinical studies · team)
   - *Default: infer from the brief; if unclear, (c) authority page.*

2. **Primary audience for this page?**
   - Surgeons/HCPs · device manufacturers · researchers · public institutions/funders · prospective trainees · general/mixed
   - *Default: the audience the page topic most serves (e.g. training → prospective trainees).*

3. **Key messages + which proof points are TRUE and usable here?**
   - List the 2–4 things this page must land, and tick the proof points that apply
     (trained +1150 since 2018 · +150 explants in 2023 · +20 studies · ISO 9001/13485 · Qualiopi · world's largest vascular-explant database · since 1993).
   - *Default: anchor to the page's pillar; use only brand-kit proof points that obviously apply; bracket anything new.*

4. **SEO / GEO target keyword(s)?**
   - One primary query + 2–3 secondary keywords the page should rank/answer for.
   - *Default: infer 1 primary + 2–3 secondary from the topic and audience.*

5. **Structure & length?**
   - Use the standard page-type skeleton, or specify sections.
   - Length: concise · standard · in-depth. Language: FR · EN · mirror the brief.
   - *Default: standard length, page-type skeleton, language mirrors the brief.*

---

**When to skip intake entirely:** the brief already specifies page type, audience,
the messages/proof, and either keywords or enough to infer them. In that case,
state a one-line `Assumptions:` and generate.

**Training-page note:** for a training page, also confirm (or bracket) the
Qualiopi-required fields — objectives, target public, prerequisites, duration,
modalities (in-person/simulation), accessibility/handicap contact, price &
financing, and published satisfaction/success indicators. Missing values are
`[bracketed]`, never invented.
